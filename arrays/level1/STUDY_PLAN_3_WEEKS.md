# Arrays — 3-Week Solving Plan

91 problems across 21 days (~4–5/day). Order builds skill-on-skill: foundations →
windows & search → extras, with new mental models isolated and a review day each week.

For each problem: solve the 🟢 to lock the template, then 🟡 → 🔴. Re-solve anything
you needed a hint on the next day. Aim for a clean medium in ~20 min by week 3.

IDs map to the method comments in code (see `Arrays_Patterns_Detailed.md`).

## Pace Rules
- Stuck > 25–30 min? Read the approach, understand it, then re-implement from scratch.
- Always state brute force → optimal, with time/space, before coding (interview habit).
- Track which problems needed hints — those are your Day 7 / 14 / 20 review list.
- If a day runs long, push the last extra-topic item, never a roadmap-core item.
